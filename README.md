# Secure Authentication System (C/Unix)

**Безопасная система аутентификации на C с разделением на HTTPS-сервер, изолированный демон и базу данных PostgreSQL.**

###  Цель проекта
Разработка отказоустойчивого решения для проверки паролей, демонстрирующая работу с системными вызовами Unix, низкоуровневую работу с сетью и принципы обеспечения безопасности.

###  Архитектура (Client-Server + Daemon)

Система состоит из 3 независимых модулей, взаимодействующих через Unix Domain Sockets:

1.  **HTTPS Server (`server.c`)**:
    - Обрабатывает входящие HTTPS-соединения (OpenSSL).
    - Вычисляет SHA-256 хеш пароля перед отправкой демону.
    - Реализует Zero-copy передачу статических файлов (`index.html`, `login.html`) через `mmap`.
2.  **Auth Daemon (`auth_daemon.c`)**:
    - Работает как фоновый процесс (реализован через двойной `fork()` и `setsid()`).
    - Использует неблокирующие сокеты и `poll()` для одновременной обработки до 10 клиентов.
    - Выполняет строгую валидацию входных данных для защиты от переполнения буфера.
3.  **Database Module (`database.c`)**:
    - Интеграция с PostgreSQL через интерфейс `libpq`.
    - Использование параметризованных запросов для защиты от **SQL-инъекций**.

###  Запуск и проверка

1. **Установка зависимостей (Ubuntu/Debian):**
```bash
make install_deps
sudo -u postgres psql -c "CREATE DATABASE auth_system;"
psql -d auth_system -f schema.sql
```

2. **Генерация SSL сертификатов:**
```bash
make certs
```

3. **Сборка и запуск:**
```bash
make all

# Терминал 1: Запуск демона аутентификации
./auth_daemon

# Терминал 2: Запуск HTTPS сервера
./server
```

4. **Проверка:**
Откройте браузер по адресу `https://localhost:8443`.
*Примечание: так как сертификат самоподписанный, необходимо подтвердить исключение безопасности в браузере.*

**Тестовые данные:**
- **Логин:** `alice`
- **Пароль:** `qwerty123`

###  Структура проекта
*   `server.c` — Логика HTTPS-сервера.
*   `auth_daemon.c` — Код фонового процесса аутентификации.
*   `database.c / .h` — Модуль взаимодействия с БД.
*   `schema.sql` — Схема базы данных.
*   `Makefile` — Сценарий автоматизированной сборки.
*   `html/` — Директория со статическими файлами интерфейса.
